# Security Policy

Repo Doctor AI is a local CLI and does not require an API key for its deterministic scan workflow.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately by opening a GitHub security advisory or contacting the maintainers through the project issue tracker with a minimal, non-public description.

Include:

- Affected version or commit
- Reproduction steps
- Expected and actual behavior
- Any relevant environment details

Please do not publish exploit details until maintainers have had time to review and respond.

## Scope

Security reports are most useful when they affect:

- Local file handling
- Template generation safety
- CLI argument handling
- Dependency or packaging behavior

Repo Doctor AI is not a replacement for dependency audit tools or deep vulnerability scanners.
